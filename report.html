Introduction

My website is a portfolio of work that I have done in the past 5 years, extracurricular or otherwise. I chose to display this information about my work in multiple separate pages that are accessed through a unique drag and drop menu on the index page.

Inspirations

I was inspired to implement the drag and drop menu after being introduced to jQuery UI in one of Dr. Sarah Wiseman’s lectures. Examples of the draggable and droppable classes were shown, and I was curious to see how I could base a whole website around that functionality.

The styling of the blue, red, green and pink project icons on the home page was inspired by the buttons on a Dualshock controller from all of the iterations of the PlayStation console, which have the same colours.

The layout of my project pages was inspired by the layout of the game pages on the Steam website, where two thirds of the page space are dedicated to videos/photos of the games, while a third is dedicated to a writeup about that game.

Accessibility

All of the images on my site have alt tags, meaning that any text to speech software used by people with visual impairments will be able to pick up and read a description of the images.

In one case on my website, I have used an abbreviation to save space. I made sure to put periods between each letter in this abbreviation to make sure that text to speech software would read it out properly.

I have made sure that all text on the website is either black and placed directly over a light grey or white background or white and placed over a vibrant colour that isn’t too bright. I took screenshots of my pages and converted them to greyscale to make sure that text was still easy to read for people who are colour-blind.

Usability

If none of the JavaScript on any of the pages works, this will result in the drag and drop menu not working. However, in this situation the website could still be easily navigated using the links in the footer of the page.

The images on the project pages only take up the whole page when the user clicks or taps on them to enlarge them. This is especially useful on mobile, where there isn’t a lot of screen real estate to work with.

The high resolution versions of images on my websites are not automatically displayed upon a page loading. Instead, they are initially represented by much lower resolution thumbnails. This reduces the amount of time it takes for the page to load, as well as reducing unnecessary data use on mobile.

What I learnt

Before this project, I had never used jQuery UI before. I had to learn about it to be able to implement the drag and drop code on my website. Upon learning about it, I also found out how to do object fade in/fade out transitions, which I used to polish the site.

I learnt a lot about CSS positioning, which I have found very frustrating in the past. I can now make a website and know that I can position elements side by side, in grids, or at the centre of the page with some confidence.

I learnt how to dynamically reposition HTML elements at runtime using JavaScript with jQuery. This was used to make the spinning icons effect on the homepage.

What went well

I managed to fully realise my drag and drop menu, creating custom JavaScript code to make the project icons rotate around the centre of the page when not being dragged and follow the mouse when being dragged.

I think my use of jQuery UI animations isn’t obtrusive or obnoxious, and improves the interactivity and presentation of the website as a whole (e.g. fading in project thumbnail when dragging an icon over the drop area).

The colour scheme and layout of my website ensures that the most important content on a page is the largest and most prevalent, while also making sure that text content is well organised and easy to read.

What I'd improve

I would have liked to give all of the pages more interesting backgrounds that were relevant to the individual projects. For instance, I would have liked to have a space themed background on the Disasteroids page, or an anatomy themed background on the Interactomy page.

I have used flexbox and CSS grid throughout a lot of my project pages, which are both relatively new technologies that may not be supported on a lot of older browsers. I would have liked to find a way to recreate the same results I got, but with CSS techniques that are more widely supported.

There is a video on my website that contains dialogue, but does not have subtitles. This makes that content inaccessible for people who have problems with hearing. In the future, I would ensure that any YouTube content has the appropriate subtitles or closed captions.
